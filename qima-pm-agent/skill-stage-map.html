Stage Mapping

1

Intake · Raw Data Collection

Collect original demand, meeting notes, SOPs, examples, workflow diagrams, and stakeholder context.

Agent
PM Agent

PM Agent digests raw materials and can also guide the user through conversational intake to fill missing context and open questions.

2

PRD / Design · Requirements Output

Produce PRD, UI direction, interaction notes, and stakeholder-readable requirement artefacts.

Agent
PM Agent

PM Agent turns raw data into a PRD, early design intent, and prototype views for stakeholder confirmation.

3

Grooming · Scope Clarification

Clarify scope, review feasibility, refine stories, and surface risks before sprint commitment.

Agents
PM Agent Developer Agent QA Agent

PM Agent iterates PRD and ticket breakdown; Developer Agent checks feasibility and workload; QA Agent reviews testability and validation risk before sprint commitment.

4

Planning · Sprint Commitment

Confirm sprint backlog, user stories, feasibility, workload, and team commitment.

Agents
PM Agent Developer Agent QA Agent

PM Agent completes Epic and User Story breakdown; Developer Agent confirms workload and implementation approach; QA Agent confirms test planning needs.

5

Development · Delivery Execution

Build the solution, prepare implementation details, and open PRs for review.

Agent
Developer Agent

Developer Agent owns implementation work: understand ticket context, propose technical design, write code, generate tests, and prepare PR handoff.

6

Code Review · PR Quality Gate

Review implementation with AI assistance and human approval before merge.

Agent
Developer Agent

Code review stays within the developer workflow. It is a review capability, not a standalone management-level agent in this story.

7

QA · Functional + Regression Testing

Test functionality, integration path, regression scope, and bug readiness.

Agent
QA Agent

QA Agent generates test cases from acceptance criteria, supports regression thinking, and prepares bug-ready findings when behaviour fails.

8

PM Acceptance · Consistency Validation

Validate whether delivery matches PRD, UI design, main flow, behaviour, and expected logic.

Agent
PM Agent

PM Agent supports high-level PM review: does the delivered product match what the PM requested?

9

Release · Production Readiness

Move from staging to production with release checks, security review, and operational confidence.

Agents
Developer Agent DevOps Agent Security Agent

Developer Agent can drive basic release readiness; DevOps and Security Agents are planned extensions for deployment and compliance checks.

10

Retro · Delivery Review

Review delivery result, risks, collaboration issues, and improvement proposals.

Agents
PM Agent Developer Agent QA Agent

The core trio reviews what went well, what created rework, and what should be improved in the next intake-to-release cycle.

Agent Capability Map

Primary agents and their underlying skills

The top layer is deliberately simple: PM, Developer, and QA. More specialised agents can be introduced later as the workflow matures.

PM Agent

qima-pm-agent
Primary

Turns raw product data into PRD, design intent, Epic and User Stories, then supports PM acceptance by checking delivery consistency against PRD and UI expectations.

research codebase-understanding write-prd prd-critique ticket-breakdown write-ticket prd-design-assets design-qa-review
Covers stages 1-4 and 8; owns product intent and handoff quality.

Frontend Developer Agent

frontend-developer-agent
Primary

Reads tickets, analyses requirements, proposes front-end approach, implements Angular / Nx changes, writes unit and E2E tests, and prepares PR workflow.

analyse-requirement implement-feature angular-unit-test cypress-e2e-test git-pr-workflow
Covers stages 3-6; owns front-end feasibility, delivery, and review handoff.

Backend Developer Agent

backend-dev-agent
Primary

Reviews Jira context, looks up QSP architecture, prepares technical design, implements backend changes, traces service context, and opens PRs.

jira-ticket-review technical-design code-implementation code-trace open-pr
Covers stages 3-6; owns backend feasibility, delivery, and review handoff.

QA Agent

qa-agent
Near term

Generates test cases from acceptance criteria, supports functional and regression validation, and turns observed failures into bug-ready descriptions.

qa_analyze qa_create_testcase qa_run_testcase
Covers stages 3, 4, 7 and 10; owns testability and validation readiness.

Expert Review Capability

review-skill
Embedded

Treat review as a capability inside PM or Developer workflows, not a standalone management-level agent.

DevOps Agent

devops-agent
Planned

Deployment checks, environment validation, CI/CD pipeline support, monitoring and release readiness.

Security Agent

security-agent
Planned

Security review, vulnerability assessment, compliance checks and remediation guidance before release.

Analytics Agent

analytics-agent
Planned

Agent health metrics, R&D efficiency tracking, rework analysis and monthly progress reporting.

Data Agent

data-agent
Planned

SQL assistance, dashboards, reporting and data pipeline support across QIMA platforms.
