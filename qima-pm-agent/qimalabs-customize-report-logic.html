How QIMAlabs Already Supports Report Customisation

QIMAlabs already has a report generation foundation: report pages, templates, result summary controls, customised sections, online editing, preview, and final report generation. The new customisable report feature should extend these existing capabilities instead of rebuilding the report flow from scratch.

Product Logic

How to think about the feature

Template Group is the setup layer

  • Defines which Program uses which report template.
  • Supports language-specific report templates.
  • Uses tags to decide where sections such as Result Summary or Specimen Description are inserted.
  • Should become the configuration foundation for the future customisable report feature.

Customised Section is the execution layer

  • Lets users manage report-specific custom content.
  • Allows manual editing, system auto-fill, and semi-auto fill with user confirmation.
  • Works close to the actual report, which is important for First Page-level isolation.
  • Can provide the user-facing entry point for custom fields, sections, order, and file editing.

Risks and Decisions

What must be clarified before delivery

Highest product risk

Standard reports must continue to behave exactly as they do today when no custom setup is configured.

  • Customisation must be isolated per First Page. One FP must not change another FP's report.
  • Manual edits and auto-generation need clear overwrite rules.
  • Missing template language should raise a clear setup issue, not silently fall back.
  • Missing section tags in templates should be visible to users or admins.
Product decisions needed

The first release should define a narrow and testable scope before expanding to full report layout customisation.

  • Decide which sections are P0: Result Summary, Specimen Description, Additional Section, Failure Content, or all of them.
  • Define who can edit, auto-fill, preview, approve, and generate the final report.
  • Clarify whether users customise at Program level, order level, FP level, or section level.
  • Confirm how customised reports should behave with e-signature, final PDF, and report sending.
